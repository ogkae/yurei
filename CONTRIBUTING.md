## Contributing to Yurei // Toaster License 2025

Thank you for considering contributing to Yurei. This project provides cryptography utilities, ID generators, obfuscation helpers, and a lightweight key-value store under the humorous **Toaster License 2025**. We welcome contributions while keeping the project fun and educational.

---

## How to Contribute

### 1. Star the Repository
Supporting the project with a star costs nothing and helps others discover it.

### 2. Report Bugs
- Include a detailed description of the issue.
- Mention your environment and Python version.
- Screenshots or short reproducible examples are highly appreciated.
- Note: Issues caused by your “toaster PC” or copied tutorials from YouTube skids are considered part of the fun and may be ignored.

### 3. Suggest Features
- Feature requests should improve usability, clarity, or add educational value.
- Humorous or educational references to “novice hackers copying GPT” are welcome.
- Examples: deterministic HMAC ID generators, parallel encryption helpers, or enhanced obfuscation utilities.

### 4. Fix Typos or Improve Documentation
- Clear, concise, and grammatically correct documentation is always appreciated.
- Adding contextual examples or clarifying comments in code is encouraged.

### 5. Submit Pull Requests
- Fork the repository and create a new branch for your changes.
- Keep commits small and descriptive.
- Include tests where applicable.
- Maintain code readability and follow PEP8 style guidelines.
- Ensure PRs do not break existing functionality or security features.

---

## Code of Conduct
- Be respectful to all contributors, including those who learned Python from YouTube skids tutorials.
- Keep humor in good taste. This is a fun project, not a place for harmful content.
- Violations of the Code of Conduct may result in your PR being politely ignored.
